Getting started

Go from download to your first classified dataset in under 10 minutes.

0 Prerequisites

Before you open the app, you'll need API keys for the services it calls. The app itself is free — you only pay for actual API usage.

Required

OpenRouter API Key

Routes requests to Claude, GPT, and other LLMs. Sign up at openrouter.ai and create an API key. Pay-as-you-go, typically a few cents per classification batch.

Optional

Exa API Key

Powers automatic web research on unknown suppliers. Sign up at exa.ai. Without this, classification still works but without supplier research context.

1 Install the app

  1. Download the .dmg file from the Download section
  2. Double-click the DMG to mount it
  3. Drag Colab91 Classifier into your Applications folder
  4. On first launch, macOS may show a security warning — go to System Settings → Privacy & Security and click "Open Anyway"
  5. The app opens to the project screen — you're ready to go
  1. Download the .exe installer from the Download section
  2. Run the installer — Windows SmartScreen may prompt you; click "More info" then "Run anyway"
  3. Follow the installation wizard (default settings are fine)
  4. Launch Colab91 Classifier from the Start Menu or desktop shortcut

2 Configure your API keys

  1. Click Settings (bottom-left corner of the sidebar)
  2. Under LLM Configuration, select OpenRouter as the provider
  3. Paste your OpenRouter API key — it's encrypted and stored locally via Electron's safeStorage
  4. (Optional) Under Web Search, paste your Exa API key for supplier research
  5. The default model is claude-3-5-haiku-latest — you can change it to any model available on OpenRouter
Tip

Your API keys never leave your machine. They're encrypted at rest and sent directly to the API providers — no middleman server.

3 Create your first project

  1. Click New Project on the welcome screen
  2. Choose a name and save location — this creates a .c91 file (a self-contained DuckDB database)
  3. Fill in Company Context: your company name, industry, and a brief description. This context helps the AI make smarter classification decisions

4 Import your transaction data

  1. Navigate to Data in the sidebar and click Import CSV
  2. Select your AP transactions CSV file
  3. The app's Column Mapping Agent (an LLM) automatically maps your CSV columns to canonical fields: supplier_name, spend_amount, gl_description, invoice_number, etc.
  4. Review the suggested mapping and correct any that look off, then confirm
  5. The Data Review page shows all imported transactions — filter, sort, and inspect before moving on
Tip

Column mappings are remembered per upload. The next time you import a CSV with the same format, the mapping is applied automatically.

5 Set up your taxonomy

The taxonomy defines how transactions get classified. It's a 3-level hierarchy:

  • L1 (Category) — e.g., "Corporate Services"
  • L2 (Sub-category) — e.g., "Information Technology"
  • L3 (Detail) — e.g., "Software Licenses"

Paths are displayed as pipe-delimited strings: Corporate Services|IT|Software

Option A: Import from ColabIP

If you have a master taxonomy file (CSV/Excel), import it from the Taxonomy page. The app parses the L1/L2/L3 columns automatically.

Option B: Build from scratch

Use the Taxonomy editor to add categories manually. You can always edit, add, or remove entries later.

6 Run classification

  1. Navigate to the Classification page
  2. Select which transactions to classify (all, unclassified only, or a filtered subset)
  3. Click Start Classification — the progress bar shows real-time status

Behind the scenes, the app runs a multi-step pipeline:

  • Supplier Research — unknown suppliers are researched on the web via Exa to understand what they sell or do
  • Override Rules — if you've configured rules (e.g., "all transactions from Microsoft go to IT|Software"), matching transactions are classified instantly without using LLM credits
  • LLM Classification — remaining transactions are sent in batches to the LLM with your taxonomy, company context, and any supplier research results as context
  • QC Agent — a separate AI agent reviews the classifications and flags low-confidence or inconsistent results
Note

Classification uses your OpenRouter credits. Cost depends on the model and number of transactions. Claude 3.5 Haiku is the default — fast and cost-effective. A batch of 1,000 transactions typically costs under $1.

7 Review and correct

  1. Go to the Review page to see all classified transactions
  2. QC-flagged items are highlighted — review these first
  3. Click any transaction to see the AI's reasoning and confidence score
  4. Correct misclassifications by selecting the right L1/L2/L3 from the taxonomy dropdown
  5. Your corrections feed back into the system — the Feedback Agent learns from corrections to improve future runs

8 Export results

  1. Navigate to Output in the sidebar
  2. Choose your export format (CSV)
  3. Select which columns to include in the export
  4. Click Export and choose a save location
Tip

Your project .c91 file retains all data, classifications, and history. You can re-open it anytime, import new batches, re-run classification, or share the file with a colleague.

Under the hood

A look at the technical architecture for the curious.

Renderer (React + Zustand)  <-- IPC -->  Main Process (Node.js)
                                             ├── DI Container (createContainer)
                                             ├── IPC Handlers
                                             ├── Orchestrators
                                             │   ├── ClassificationRunner
                                             │   ├── SupplierResearcher
                                             │   └── QCRunner
                                             ├── 6 LLM Agents
                                             │   ├── ClassificationAgent
                                             │   ├── ResearchAgent
                                             │   ├── ColumnMappingAgent
                                             │   ├── FeedbackAgent
                                             │   ├── QCAgent
                                             │   └── CatAAgent
                                             ├── Data Services (stores, ingest, export)
                                             └── Leaf Layer
                                                 ├── DuckDB
                                                 ├── LLMClient → OpenRouter
                                                 └── ExaWebSearch → Exa API

Electron + React

Main process handles all backend logic, DB access, and LLM calls. Renderer is a React SPA with Zustand state management, connected via typed IPC channels.

DuckDB Storage

Each project is a single .c91 DuckDB file — embedded, zero-config, portable. Dynamic schema: canonical columns are added at runtime as CSVs are imported.

Dependency Injection

All services receive dependencies via constructor — no module-level singletons. A lightweight DI container wires everything in dependency order: Leaf → Agents → Services → Orchestrators.

6 Specialized Agents

Each agent has a focused job: classify transactions, research suppliers, map columns, incorporate feedback, run QC, and handle category analysis. Pure prompt builders + parsers make them testable.

EventBus

A typed event emitter handles async progress (classification %, supplier research status). Services emit events; the entry point forwards them to the renderer via webContents.send.

Supplier Universe

A shared DuckDB file stores global supplier data, aliases, and research info across projects. FK-based model: transactions reference suppliers via foreign key, not string matching.

Frequently asked questions

Is the app free?

Yes, the app itself is completely free. You only pay for LLM API usage through OpenRouter (pay-as-you-go). A typical batch of 1,000 transactions classified with Claude 3.5 Haiku costs under $1.

What format does my CSV need to be in?

Any CSV with transaction data works. The app uses an AI-powered column mapping agent that automatically detects and maps your columns to canonical fields (supplier name, amount, GL description, etc.). You review and confirm the mapping before import.

Which LLM models are supported?

Any model available on OpenRouter works — including Claude (3.5 Haiku, Sonnet, Opus), GPT-4o, Gemini, Llama, Mistral, and more. The default is Claude 3.5 Haiku for the best balance of speed, accuracy, and cost. You can change the model in Settings.

Is my data sent to the cloud?

Transaction data is sent to the LLM provider (via OpenRouter) only during classification — this is necessary for the AI to classify your transactions. All data is stored locally on your machine in DuckDB files. API keys are encrypted via Electron safeStorage. No data is sent to any other service.

Can I share a project with a colleague?

Yes! Each project is a single .c91 file. Send it to a colleague and they can open it directly in the app. All transactions, classifications, taxonomy, and settings are contained in that one file.

What are override rules?

Override rules let you auto-classify transactions that match specific conditions (e.g., supplier name contains "Microsoft" → IT|Software) without using LLM credits. Rules are evaluated by priority before the AI runs, so known patterns are handled instantly and for free.

Does the app work offline?

You can open projects, browse data, and export results offline. Classification requires an internet connection since it calls the LLM and web search APIs. Override rules work offline since they're evaluated locally.

How accurate is the classification?

Accuracy depends on your taxonomy complexity, company context quality, and the model you choose. With good context and a well-structured taxonomy, users typically see 85-95% accuracy on the first pass. The QC agent flags uncertain results, and your corrections improve future runs via the feedback agent.

What's the Supplier Universe?

The Supplier Universe is a shared database that stores supplier information, aliases, and web research results across all your projects. When the app researches a supplier for one project, that knowledge is available to all future projects — no duplicate research needed.
